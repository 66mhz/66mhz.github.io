System Design

2022-07-16
  • System design is about understanding the fundementals of building blocks, and piecing them together using problem-solving skills.
  • It is about showing leadership skills, showing initiativeness and discussing the trade-offs, as there is often a trade-off once a decision is made.
  • Rather than the solution, how you approach to a problem, to show problem-solving skills is what matters.
  • It is not to memorize solutions. It is about how to solve problems. It is about:
    • Understand the requirements
    • Find solutions
    • Make decisions and discuss trade-offs.
  • Schedule a live-session mock interviews with senior engineers.
  • There is a typical flow to it.
    • Gather requirements. Clarify the requirements, establish baseline and assumptions made.
      • Who is this for? What are the features we need?
    • Design API
      • Do QPS if needed
    • Draw high level diagrams
    • Schema and data structures
      • Optional storage calculation
    • End to end flow summary
    • Maybe some deep dives
